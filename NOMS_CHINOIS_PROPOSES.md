# Noms chinois proposés — provinces ajoutées à Ming sans nom chinois

> Provinces que **le mod** rattache à MNG (owner = MNG dans le mod, mais ≠ MNG dans le jeu de base) **et** qui n'ont **pas** encore d'entrée dans `common/province_names/east_asian.txt`.
>
> Objectif : leur donner un **nom chinois réaliste, historique et plausible** (à terme, une entrée `id = "Nom"` dans un `province_names/east_asian.txt` du mod), pour que Ming affiche un toponyme chinois.
>
> Méthode de nommage : priorité aux **noms administratifs de l'époque Ming/Qing** (gardes/衛 Ming, préfectures, postes et bannières Qing), qui sont les exonymes chinois historiques de ces lieux. Romanisation **pinyin** (style cohérent avec `east_asian.txt` : « Gaoping », « Haifang »…), avec caractères 漢字.
>
> **Confiance** : `Haute` = nom chinois historique attesté · `Moyenne` = toponyme chinois plausible/dérivé · `Basse` = translittération phonétique faute de source ferme (à vérifier en priorité).
>
> Total : **76 provinces** · à valider au cas par cas (colonne *Validé ?*).
> Généré pour `Chinese_Culture_and_Religion` — EU4 v1.37.5.0.

## Notes méthodo

- Les noms `Haute` confiance sont des **toponymes chinois attestés** (gardes Ming 衛, préfectures, postes/bannières Qing) — peu de risque.
- Les `Moyenne` sont des formes chinoises plausibles mais à confirmer (orthographe des caractères surtout).
- Les `Basse` sont des **translittérations phonétiques** faute de nom historique clair (surtout Sibérie/Extrême-Orient et toponymes russes) — **vérifie-les en priorité**, ou indique si tu préfères les laisser sans nom chinois.
- Une fois validés, ces noms iront dans un fichier `common/province_names/east_asian.txt` du mod (format `id = "Nom"`), qui **surcharge** celui du jeu de base — il faudra alors **recopier aussi les entrées existantes** du fichier de base pour ne pas les perdre.
